익명함수에 대해 알아봅시다.

프로그램에서 먼저 사용되지 않을 함수가 엄청(10,000개) 많을 때 익명함수 표기법을 사용해서 그 함수를 사용할 때만 함수를 메모리에 저장하는 방법

  • function name(){} : 함수(메모리에 먼저 저장)
  • setInterval(name,1000) : 미리 선언한 함수를 1초마다 호출
  • **setInterval(function(){},1000) : 익명함수를 매개변수로 사용한 예) 1초마다 익명함수가 호출되기 때문에 콜백함수(call back function)라 한다.
  • function(){} : 익명함수(이름이 없는 함수, 매개변수로 바로 정의 후 대입)
  • const name=function(){} : 익명함수 표기법(함수 표현식)
  • 익명함수 표기법을 사용하면 함수의 호이스팅을 막을 수 있기 때문이다.
  • 함수의 호이스팅 : 아래에 정의한 함수가 위에서 호출가능한 상태(자바스크립트 엔진이 함수를 먼저 찾아서 메모리에 저장하기 때문. 이 때 익명함수는 제외)
